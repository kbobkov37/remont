**Промпт для разработки приложения "Ремонт оборудования" с использованием PostgreSQL**

---

**Описание проекта:**

Создать веб-приложение "Ремонт оборудования", которое позволяет управлять задачами ремонта, оборудованием, 
клиентами и исполнителями. Использовать PostgreSQL в качестве базы данных. Приложение должно обеспечивать 
CRUD-операции для всех основных сущностей и иметь простую интерфейсную часть.
В приложении должны быть реализованы следующие функции:
Регистрация пользователей с проверкой.
Авторизация пользователей с проверкой.
Возможность добавлять, редактировать и удалять записи о ремонтах и комплектующих через формы.
Отображение связанной информации: какие комплектующие использовались при ремонте.
Просмотр истории ремонтов и движения комплектующих.
Реализация CRUD-операций для ремонтов и комплектующих.
Вывод данных в таблицы или списки с возможностью фильтрации.
Окна для отображения списка ремонтов и комплектующих.
Чистая архитектура — разделение логики базы данных и интерфейса.
Инструкции по запуску и использованию.
Чистый, хорошо структурированный код с комментариями.
---

**Требования к базе данных:**

-- 1. Таблица: Сотрудники
employees (
    employee_id -- Уникальный идентификатор
    full_name  -- ФИО, не может быть пустым
    position  -- Должность
    email -- Email должен быть уникальным
    phone -- Номер телефона сотрудника
);

-- 2. Таблица: Тип платы
boards_type (
    board_type_id -- Уникальный идентификатор
    board_type_name -- Название (например, 'ИСТ-002')
    description -- Подробное описание
);

-- 3. Таблица: Тип оборудования
equipments_type (
    equipment_type_id -- Уникальный идентификатор
    equipments_type_name -- Название типа (например, 'Таксофон')
);

-- 4. Таблица: Вендор (Производитель)
vendors (
    vendor_id -- Уникальный идентификатор
    vendor_name -- Название производителя (например, 'ИСиТ')
    support_url -- Ссылка на техподдержку
);

-- 5. Таблица: Контрагент (Заказчик)
contractors (
    contractor_id -- Уникальный идентификатор
    company_name -- Название контрагента (например, 'ИжСервис')
    addres -- Адрес контрагента
    contact_person -- Контактное лицо
    phone -- Контактный телефон
);

-- 6. Таблица: Тип ремонта
repairs_type (
    repair_id -- Уникальный идентификатор
    repair_name -- Тип ремонта, (Ремонт своими силами, ремонт у контрагента)
);

-- 7. Таблица: Склад
store (
    store_id -- Уникальный идентификатор
	part_id -- Уникальный идентификатор детали
    store_quantity -- Количество деталей с идентификатором part_id на складе
);

-- 8. Таблица: Детали
parts (
    part_id -- Уникальный идентификатор детали
    part_type -- Тип детали (например, 'Микросхема')
    part_name -- Название детали (например, 'DS1307')
	body_id -- Уникальный идентификатор корпуса
    part_quantity -- Количество
    part_cell_id -- Место для хранения детали
    part_price -- Цена детали
    part_date -- Дата записи
	part_pdf -- Документация для детали, адрес расположения файла на компьютере
	part_url -- Документация для детали, интернет адрес даташита
);

-- 9. Таблица: Корпуса
bodys (
    body_id -- Уникальный идентификатор корпуса
    body_name -- Название корпуса детали (например, 'SOIC-8')
	body_pic -- Картинка детали, адрес расположения файла на компьютере
);

-- 10. Таблица: Статус ремонта
status (
    status_id -- Уникальный идентификатор корпуса
    status_name -- Название корпуса детали (например, 'SOIC-8')
);

-- 11. Таблица: Ремонт
repairs (
    repair_id -- Уникальный идентификатор
	equipment_type_id -- Тип оборудования
	board_type_id -- Тип платы
	date_repair_in -- Дата поступления в ремонт
	date_repair -- Дата ремонта
	date_repair_out -- Дата выдачи
	status_id -- Статус платы (неисправна, отремонтирована, выдана)
);
******************************************************************

type_chips (

    type_chip_name -- Название корпуса детали (например, 'SOIC-8')
	type_chip_pic -- Картинка типа детали, адрес расположения файла на компьютере

    body_id -- Уникальный идентификатор корпуса

	body_pic -- Картинка корпуса детали, адрес расположения файла на компьютере
);
-- 3. Таблица: Документация для детали, адрес расположения файла на компьютере
chip_pdfs (
    chip_pdf_id -- Уникальный идентификатор корпуса
    chip_pdf_name -- Адрес расположения файла на компьютере
);
-- 4. Таблица: Документация для детали, интернет адрес даташита
chip_urls (
    chip_url_id -- Уникальный идентификатор
    chip_url_name -- интернет адрес документации для детали
);
-- 5. Таблица: Детали
parts (
    chip_id -- Уникальный идентификатор детали
    part_type_id -- Уникальный идентификатор типа детали (например, 'Микросхема')
    chip_name -- Название детали (например, 'DS1307')
	body_id -- Уникальный идентификатор корпуса
    chip_quantity -- Количество
    chip_cell -- Место для хранения детали
    chip_price -- Цена детали
    chip_date -- Дата поступления детали
	chip_pdf_id -- Документация для детали, адрес расположения файла на компьютере
	chip_url_id -- Документация для детали, интернет адрес даташита
);

**Функциональные требования:**

- Регистрация и редактирование данных оборудования, клиентов, специалистов, ремонтных заказов.
- Просмотр списка всех записей с возможностью фильтрации по статусу, дате и другим параметрам.
- Возможность добавлять новые записи и обновлять существующие.
- Удаление записей при необходимости.
- Простая панель управления.

---

**Технические детали:**

- Использовать PostgreSQL для хранения данных.
- Backend – Python.
- Фронтенд – фреймворк(Django или FastAPI).
- Использовать SQLAlchemy для взаимодействия с базой данных.
